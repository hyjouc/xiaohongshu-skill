# Anti-AI Xiaohongshu Copy Checklist

Use this before writing and again before filling a Xiaohongshu draft.

## What Makes Copy Feel AI-Written

- Abstract conclusions without scenes: "提升效率", "形成闭环", "持续优化", but no real user action.
- Too-even structure: every section has the same length, same rhythm, same "解释 + 总结" pattern.
- Over-polished transitions: "首先/其次/最后", "综上", "值得注意的是", "核心在于".
- Product-brochure tone: everything is useful, important, recommended, with no doubt or tradeoff.
- Fake certainty: claims sound universally true, but no boundary or context is named.
- Mechanical list density: too many bullets, too many numbered sections, no sentence-level variation.

## Human-Sounding Inputs To Invent Carefully

If the user did not provide lived experience, do not fake a personal story. Instead use a neutral observed scenario:

- "我看到很多人会卡在..."
- "这类任务最容易出问题的地方是..."
- "如果你只是想先跑通，可以先..."
- "我不建议一开始就..."

Use one small imperfection or tradeoff:

- "这个方法不酷，但省返工。"
- "它不适合所有内容，尤其是需要强观点的选题。"
- "别一上来追全自动，先让流程跑顺。"

## Rewrite Rules

- Replace slogans with actions: "提升效率" -> "少来回改两三轮".
- Replace big nouns with concrete verbs: "流程优化" -> "先固定标题、正文、封面这三步".
- Replace perfect structure with natural grouping: 3-4 meaningful sections is usually enough.
- Keep some sentence length variation. Mix 8-15 character short lines with 25-40 character explanation lines.
- Add one honest boundary near the end. Xiaohongshu users trust caveats more than universal promises.
- End with a real question or lightweight prompt, not a motivational quote.

## Red Flags To Remove

Remove or rewrite these unless quoted from source:

## Final Pass

Before upload, ask:

- Is there at least one concrete user action?
- Is there one thing the writer does not recommend?
- Do any two sections start the same way?
- Could a reader tell this was written for this specific topic, not any AI/productivity topic?
- Would removing one paragraph make the note sharper? If yes, remove it.
